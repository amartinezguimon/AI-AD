# data/

Training data for the engagement model. Versioned in git (the CSVs are small).

- `raw_sessions/` — one CSV **per collection session** (one subject + one condition
  combo), written by the collector. This is the source of truth; commit new
  sessions here.
- `engagement_dataset.csv` — the **master** dataset, built by merging everything in
  `raw_sessions/` (+ the legacy file). Generated by
  `python -m visionmetrics.training.build_dataset`. This is what training reads.
- `engagement_data.csv` — the original/legacy flat dataset (4 columns). Folded into
  the master automatically.

Everything else in this folder (`*.sqlite`, `*_log.json`, `live_stats.json`, …) is
**agent runtime output**, not dataset, and is git-ignored.

## Schema (per row)
`yaw, pitch, distance, label, distance_tier, glasses, headwear, subject, collector, session, captured_at`

The model trains only on `yaw, pitch, distance` → `label` (1 = looking, 0 = away).
The rest is metadata: it tracks **coverage** (do we have glasses/cap/far examples?)
and lets training report accuracy **per condition**. See `../visionmetrics/training/README.md`.
